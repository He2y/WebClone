# DJI 官网复刻项目

基于 DJI 官网的前端复刻项目，实现了完整的电商购物流程。

## 📋 项目特性

- ✅ **静态页面**：不依赖后端 API，所有数据写在前端代码中
- ✅ **默认登录状态**：显示已登录用户状态，包含购物车等信息
- ✅ **完整交互**：搜索、筛选、排序、添加购物车等功能可用
- ✅ **响应式设计**：支持桌面端和移动端浏览
- ✅ **无外部依赖**：图片、字体等资源本地化（图片使用占位符）
- ✅ **多页面**：首页、搜索、详情、购物车、结算等核心页面

## 🚀 快速开始

### 1. 克隆或下载项目

```bash
cd E:\webClone\Dji.com
```

### 2. 使用本地服务器运行

由于使用了 ES6 模块（type="module"），需要通过 HTTP 服务器运行，不能直接用 file:// 协议打开。

#### 使用 Python 启动服务器

```bash
# Python 3
python -m http.server 8000

# Python 2
python -m SimpleHTTPServer 8000
```

#### 使用 Node.js 启动服务器

```bash
# 安装 http-server
npm install -g http-server

# 启动服务器
http-server -p 8000
```

#### 使用 VS Code Live Server

1. 在 VS Code 中安装 Live Server 扩展
2. 右键点击 `index.html`
3. 选择 "Open with Live Server"

### 3. 访问网站

打开浏览器访问：`http://localhost:8000`

### 4. 测试数据加载

如果页面显示空白，可以访问测试页面检查数据是否正常加载：
```
http://localhost:8000/test.html
```

测试页面会显示：
- 数据加载状态
- 产品总数
- 所有产品列表

## 🐛 常见问题

### 页面显示空白或没有产品

1. **确认使用了 HTTP 服务器**
   - ❌ 不能直接双击打开 HTML 文件（file:// 协议）
   - ✅ 必须使用 http:// 协议
   
2. **检查浏览器控制台**
   - 按 F12 打开开发者工具
   - 查看 Console 标签是否有错误
   - 常见错误：CORS 模块加载失败

3. **测试数据加载**
   - 访问 `http://localhost:8000/test.html`
   - 查看数据是否正常加载

### JavaScript 模块导入错误

如果看到类似错误：
```
Access to script at '...' from origin 'null' has been blocked by CORS policy
```

**解决方案**：必须使用 HTTP 服务器，不能直接打开文件。

## 🛠️ 技术栈

- **HTML5**：语义化标签
- **CSS3**：现代 CSS 特性、Flexbox、Grid
- **JavaScript (ES6+)**：模块化、类组件
- **LocalStorage**：购物车数据持久化（但刷新页面会恢复初始状态）

## 📊 数据管理

### 静态数据（`djiData.js`）

- **产品数据**：包含 10 款 DJI 产品（无人机、手持设备等）
- **购物车管理**：添加、删除、更新数量、计算总价
- **产品筛选**：按分类、系列筛选
- **产品搜索**：关键词搜索
- **产品排序**：按价格、名称排序

### 购物车存储

购物车数据存储在 `localStorage` 中，key 为 `dji_cart`。

**注意**：按照项目要求，刷新页面后购物车会恢复初始状态（清空）。

## 🔧 自定义配置

### 修改产品数据

编辑 `src/data/djiData.js` 文件中的 `products` 数组。

### 修改样式

编辑 `src/public/css/dji.css` 文件，支持 CSS 变量：

```css
:root {
  --color-black: #000000;
  --color-white: #ffffff;
  --color-primary: #000000;
  /* ... 更多变量 */
}
```

### 添加新页面

1. 在 `src/pages/dji/` 创建 HTML 文件
2. 创建对应的 JS 文件
3. 导入必要的组件和数据
4. 实现页面逻辑

## 📝 注意事项

1. **必须使用 HTTP 服务器**：不能直接打开 HTML 文件
2. **购物车数据**：刷新页面后会清空（符合项目要求）
3. **图片资源**：使用占位符，可自行替换
4. **浏览器兼容性**：建议使用现代浏览器（Chrome、Firefox、Safari、Edge）
